## Running locally

```bash
# Install Supabase CLI
npm install -g supabase

# Start local Supabase
supabase start

# Apply all migrations
supabase db reset

# Push migrations to production
supabase db push
```

## Applying to production (already-migrated project)

The schema described in `migrations/` has already been applied to the production Supabase project. The migration files serve as the source of truth for version control.

## Key design decisions

| Concern | Decision |
|---|---|
| Auth | Supabase Auth. `profiles` auto-created via trigger on `auth.users` insert |
| Roles | `user_roles` table with `app_role` enum (`admin`/`staff`/`customer`) |
| RLS | All tables use RLS. Anon key = storefront access. Service key (server-only) bypasses RLS |
| Products | `product_variants` table replaces flat `sizes[]` + single `stock` field |
| Orders | `order_items` supplements (not replaces) existing `items` JSONB column for backward compat |
| Loyalty | `loyalty_balances.tier` auto-recalculated from `lifetime_spent` via DB trigger |
| Announcements | `announcement_bars` replaces hardcoded strings in `AnnouncementBar.tsx` |
| Order numbers | `SD-XXXXXXXX` format generated by DB sequence default |
